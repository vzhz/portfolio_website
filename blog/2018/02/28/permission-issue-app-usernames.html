What a Permissions Issue Taught Me about Personalizing Apps

Feburary 28, 2018

Some time ago, I made a terminal app that pulls randomly from a list of encouraging phrases I'd written and prints one to my terminal when I open it in the morning. It was a bit of encouragement for my future self.


My terminal app gave me encouragement every morning and even knew my name!

This is the story of a choice I made when building it ended up causing a massive pain in my neck. The app consists of a set number of phrases, one of which is selected at random and printed to the terminal by a Python script that is run from your .bash_profile when the terminal is opened.

I had completely forgotten about this app when I was attempting to install the package manager Homebrew and my command line started insisting that I, from my Admin account, was not an admin. This was pretty puzzling since I have no account on my computer but Admin and Guest. After a great deal of Googling and crtl+F-ing (I even opened and searched the raw version of the Homebrew install for usr and USER), I remembered that I had set my USER to "veronica" in my .bash-profile so my terminal would print my name in its nicities.

That line is now safely commented out and I am able to "brew" my way to other things. My plan is eventually to go back and do what I would have liked to had the foresight to do in the first place: Set the USER to a variable in the .bash-profile instead of hard-coding it.
